# ЧАСТЬ 1. Что куда положить (картинки и музыку)

Это самая важная часть. Если файлы будут в неправильных папках или с неправильными именами — игра не покажет картинки.

## 📂 Структура проекта

После того как ты распакуешь архив, у тебя будет такая папка:

```
love-letter-app/
├── server.js
├── package.json
├── README.md  ← этот файл
└── public/
    ├── index.html
    ├── style.css
    ├── app.js
    └── assets/
        ├── cards/      ← СЮДА положи картинки карт
        ├── intro/      ← СЮДА положи 3 картинки заставки
        └── sounds/     ← СЮДА положи музыку
```

Папки `cards`, `intro`, `sounds` УЖЕ созданы внутри `public/assets/`. Ты просто кладёшь туда свои файлы.

## 🃏 Шаг 1.1 — Картинки карт

Открой папку `love-letter-app/public/assets/cards/`. Положи туда свои картинки карт и **переименуй их строго так**:

| Имя файла | Что должно быть на картинке |
|---|---|
| `1_1.png` | Обстоятельства, вариант 1 (например — дождь, фонарь) |
| `1_2.png` | Обстоятельства, вариант 2 (например — тёмный переулок) |
| `1_3.png` | Обстоятельства, вариант 3 |
| `1_4.png` | Обстоятельства, вариант 4 |
| `1_5.png` | Обстоятельства, вариант 5 |
| `2_1.png` | Свидетель, вариант 1 |
| `2_2.png` | Свидетель, вариант 2 |
| `3_1.png` | Дуэлянт, вариант 1 |
| `3_2.png` | Дуэлянт, вариант 2 |
| `4_1.png` | Защитник, вариант 1 |
| `4_2.png` | Защитник, вариант 2 |
| `5_1.png` | Палач, вариант 1 |
| `5_2.png` | Палач, вариант 2 |
| `6.png` | Заговорщик (только одна, без вариантов) |
| `7.png` | Вдова |
| `8.png` | Императрица |
| `back.png` | Рубашка карты (что видно когда карта закрыта) |

⚠️ **ВАЖНО:**
- Имена файлов пиши **точно так**, в нижнем регистре. `1_1.png`, а не `1_1.PNG` или `Card1.png`.
- Расширение `.png`. Если у тебя `.jpg` или `.jpeg` — переименуй в `.png` (даже если внутри jpg, обычно работает; но лучше реально сохранить как PNG).
- Между цифрой и подчёркиванием НЕТ пробелов.
- Если какого-то файла не будет — на этом месте просто покажется бежевая карточка с номером. Игра не сломается.

## 🎬 Шаг 1.2 — Картинки заставки

Открой `love-letter-app/public/assets/intro/`. Положи туда **3 картинки** твоей заставки и переименуй:

| Имя файла | Что это |
|---|---|
| `panel1.png` | Первая часть заставки (показывается первой) |
| `panel2.png` | Вторая часть (поверх первой по тапу) |
| `panel3.png` | Третья часть (поверх второй по тапу) |

Картинки могут быть любого размера — каждая занимает весь экран и обрезается по краям. Лучше всего вертикальные (формат телефона 9:16).

После третьего тапа — переход в главное меню.

## 🖼️ Шаг 1.3 — Фон главного меню

Открой `love-letter-app/public/assets/`. Положи туда фон меню:

| Имя файла | Что это |
|---|---|
| `menu-bg.png` | Фоновая картинка главного меню |

⚠️ Этот файл кладётся **прямо в папку `assets/`**, НЕ в `cards/`, НЕ в `intro/`.

Поверх фона будет лёгкое затемнение, чтобы текст и кнопки читались.

## 🎵 Шаг 1.4 — Музыка

Открой `love-letter-app/public/assets/sounds/`. Положи туда свой mp3:

| Имя файла | Что это |
|---|---|
| `music.mp3` | Фоновая музыка (играет на репите с момента первого тапа) |

⚠️ Имя файла должно быть **точно** `music.mp3`. Если у тебя файл называется `mytrack.mp3` или `bg_song.mp3` — переименуй в `music.mp3`.

Музыка автоматически:
- Стартует при первом тапе по заставке (раньше браузеры просто запретят)
- Играет на репите
- НЕ прерывается при переходе между экранами (меню → лобби → игра)
- Громкость по умолчанию 40% (можно поменять в `app.js`, найди строку `bgm.volume = 0.4`)

## 🔊 Звук кнопок

Звук клика по кнопкам встроен в код — его не надо генерировать. Это короткий синтетический звук, играет автоматически при тапе на любую кнопку.

# ЧАСТЬ 2. Запуск игры на компьютере (для теста)

Это нужно, чтобы посмотреть как игра выглядит, прежде чем выкладывать в интернет.

## 🛠️ Шаг 2.1 — Установить Node.js

Это программа, которая запускает сервер игры.

1. Заходи на сайт: **https://nodejs.org**
2. Видишь две большие зелёные кнопки. Жми **левую** (там написано «LTS»).
3. Скачается установочный файл. Запусти его, нажимай «Далее» во всех окнах.

Готово.

## 📁 Шаг 2.2 — Распаковать проект

1. Скачал архив `love-letter-app.zip`.
2. Распакуй его в любую удобную папку. Например: `C:\Users\ТвоёИмя\Desktop\love-letter-app` (Windows) или `/Users/ТвоёИмя/Desktop/love-letter-app` (Mac).
3. **Запомни этот путь!** Он понадобится в следующем шаге.

## ⌨️ Шаг 2.3 — Открыть терминал в папке проекта

Терминал — это окно, куда ты будешь вводить команды.

### Если у тебя Windows:

1. Открой **Проводник**, перейди в папку `love-letter-app` (которую распаковал).
2. В адресной строке Проводника (там где написан путь) **сотри весь текст и напиши** `cmd`, нажми Enter.
3. Откроется чёрное окно — это и есть терминал, и он сразу в нужной папке.

### Если у тебя Mac:

1. Открой **Finder**, перейди в папку `love-letter-app`.
2. Кликни правой кнопкой по папке.
3. Выбери **«Новое окно терминала в папке»** (если такой опции нет — иди в Системные настройки → Клавиатура → Сочетания клавиш → Службы → включи «Новый терминал в папке»).

### Если у тебя Linux:

Открой терминал, введи `cd /путь/к/папке/love-letter-app`.

## 📦 Шаг 2.4 — Установить зависимости

В открытом терминале **введи команду** (можно скопировать и вставить):

```
npm install
```

Нажми Enter. Подожди 1-2 минуты. Появится много текста, в конце увидишь что-то вроде «added 100 packages». Это значит — успешно.

⚠️ Если появилась ошибка «npm не является внутренней или внешней командой» — значит Node.js не установлен правильно. Перезагрузи компьютер и проверь снова.

## 🚀 Шаг 2.5 — Запустить сервер

В том же терминале введи:

```
npm start
```

Нажми Enter. Должно появиться:

```
✨ Love Letter сервер запущен на порту 3000
```

Это значит, сервер работает.

## 🌐 Шаг 2.6 — Открыть игру в браузере

1. Открой Google Chrome (или любой другой браузер).
2. В адресной строке введи: `http://localhost:3000` и нажми Enter.
3. Увидишь свою игру!

### Как проверить мультиплеер локально:

1. Открой ВТОРОЕ окно браузера в **режиме инкогнито** (Ctrl+Shift+N в Chrome).
2. Введи тот же адрес `http://localhost:3000`.
3. В обоих окнах нажми **«Быстрая партия»** в течение 30 секунд — попадёте в одну комнату и начнётся игра.

### Чтобы остановить сервер:

В терминале нажми **Ctrl+C** (на Mac — Cmd+C). Сервер выключится.

# ЧАСТЬ 3. Выложить игру в интернет (бесплатно)

Чтобы играть с друзьями, игра должна быть доступна по интернету. Самый простой бесплатный способ — Render.com.

## 🌐 Шаг 3.1 — Создать аккаунт GitHub

GitHub — это место, куда ты загрузишь свой код. Render будет его оттуда брать.

1. Заходи на **https://github.com** → **Sign up**.
2. Введи email, придумай пароль и username. Подтверди email.

## 📤 Шаг 3.2 — Залить проект на GitHub

Самый простой способ — без командной строки, через сайт.

1. На GitHub нажми зелёную кнопку **«New»** (создать репозиторий).
2. **Repository name** — напиши `love-letter-app`.
3. Поставь галочку **«Public»**.
4. Нажми **«Create repository»**.
5. На следующей странице ты увидишь надпись **«…or upload an existing file»** — нажми на неё.
6. Перетащи в окно **ВСЕ файлы из своей папки** `love-letter-app` (включая папку `public` целиком, со всеми твоими картинками и музыкой). НЕ перетаскивай папку `node_modules` если она у тебя есть — она огромная и не нужна.
7. Внизу нажми зелёную кнопку **«Commit changes»**.

Готово, твой проект на GitHub.

## ☁️ Шаг 3.3 — Создать аккаунт на Render.com

1. Заходи на **https://render.com** → **Get Started**.
2. Выбери **«Sign in with GitHub»** — войди через свой GitHub аккаунт.
3. Дай Render разрешение читать твои репозитории.

## 🚀 Шаг 3.4 — Задеплоить игру

«Задеплоить» = «выложить». Звучит страшно, делается за минуту.

1. На Render нажми **«New +»** (вверху справа) → **«Web Service»**.
2. Найди в списке свой репозиторий `love-letter-app` → нажми **«Connect»**.
3. Откроется форма с настройками:
   - **Name** — придумай любое (например `dark-duel`).
   - **Region** — оставь как есть (Oregon, USA — нормально).
   - **Branch** — `main` или `master` (что предложит).
   - **Build Command** — напиши: `npm install`
   - **Start Command** — напиши: `npm start`
   - **Instance Type** — выбери **«Free»**.
4. Внизу нажми **«Create Web Service»**.
5. Render начнёт сборку. Это займёт 2–5 минут. В логах ты увидишь:
   ```
   ==> Building...
   ==> Starting service...
   ✨ Love Letter сервер запущен на порту 3000
   ==> Your service is live 🎉
   ```
6. Сверху страницы будет ссылка вида **`https://dark-duel.onrender.com`** — это твой сайт. Открой её в браузере, проверь что игра работает.

⚠️ **Важно про бесплатный Render:**
Сервер «засыпает», если им никто не пользуется 15 минут. Первый заход после сна занимает ~30 секунд (надпись «Loading…»). Это нормально для бесплатного тарифа.

# ЧАСТЬ 4. Сделать Telegram бота с твоей игрой

## 🤖 Шаг 4.1 — Создать бота

1. Открой Telegram, в поиске найди **@BotFather** (это официальный бот для создания ботов).
2. Нажми **Start** или напиши `/start`.
3. Отправь команду: `/newbot`
4. BotFather спросит **«Alright, a new bot. How are we going to call it?»** — придумай имя, например: `Тёмная Дуэль`. Отправь.
5. Потом спросит username — **должен заканчиваться на `bot`**, например: `dark_duel_game_bot`. Отправь.
6. BotFather пришлёт сообщение с **токеном** (длинная строка). **Скопируй его в надёжное место** — он понадобится в будущем (но не сейчас).

## 🎮 Шаг 4.2 — Привязать игру к боту

1. В чате с **@BotFather** отправь: `/newapp`
2. Выбери своего бота из списка (нажми на него).
3. **Title** — напиши `Тёмная Дуэль`. Отправь.
4. **Description** — напиши `Карточная дуэль на двоих`. Отправь.
5. **Photo** — попросит загрузить картинку 640x360 (обложка для приложения). Сгенерируй любую и загрузи.
6. **GIF** — пропусти, отправь `/empty`.
7. **Web App URL** — отправь свою ссылку с Render: `https://dark-duel.onrender.com` (свою!).
8. **Short name** — отправь `play`.

Готово. Твоя игра в Telegram доступна по ссылке:
```
https://t.me/dark_duel_game_bot/play
```
(подставь свой username бота)

## ⚙️ Шаг 4.3 — Прописать имя бота в коде

Теперь нужно сказать игре, что username бота — твой.

1. Открой папку `love-letter-app/public/` на компьютере.
2. Открой файл `app.js` в любом редакторе (Блокнот, Notepad++, VS Code — что угодно).
3. Нажми Ctrl+F, введи в поиск: `BOT_USERNAME`. Найди эту строку:
   ```js
   const BOT_USERNAME = 'YourBotName';
   const APP_NAME    = 'play';
   ```
4. Замени `YourBotName` на свой username (без `@`). Например:
   ```js
   const BOT_USERNAME = 'dark_duel_game_bot';
   const APP_NAME    = 'play';
   ```
5. Сохрани файл (Ctrl+S).

## 🔄 Шаг 4.4 — Обновить файл на сервере

Render автоматически следит за GitHub. Поэтому нужно обновить `app.js` и на GitHub.

1. Заходи на свой репозиторий на GitHub.
2. Открой папку `public/`, нажми на файл `app.js`.
3. Сверху справа есть значок **карандашика** (Edit). Нажми.
4. Найди в коде те же строки и замени `YourBotName` на свой username.
5. Внизу нажми зелёную **«Commit changes»**.

Render через 1-2 минуты автоматически пересоберёт сайт с новой версией.

## 🎉 Шаг 4.5 — Играть с другом!

1. Открой своего бота в Telegram (по ссылке из 4.2).
2. Нажми **Open App** (или **Start**, потом «Запустить»).
3. Тапай по заставке (3 раза) → попадёшь в меню.
4. Нажми **«Пригласить друга»**.
5. Откроется диалог — выбери друга в Telegram.
6. Друг получит сообщение со ссылкой. Жмёт на неё — у него открывается игра прямо в твоей комнате.
7. Раздача карт начинается автоматически.

# 🆘 Если что-то не работает

## ❌ Картинки карт не отображаются (видны бежевые карточки с цифрами)

- Проверь имена файлов в `public/assets/cards/`. Должны быть строго: `1_1.png`, `1_2.png`, …, `5_2.png`, `6.png`, `7.png`, `8.png`, `back.png`.
- Расширение строго `.png` в нижнем регистре.
- Если файлы заливаешь через GitHub — убедись, что они залились (зайди на GitHub, открой папку `public/assets/cards/` и убедись что файлы там есть).
- Открой DevTools (F12 в браузере), вкладка **Network**, перезагрузи страницу. Найди файлы карт — они должны быть со статусом 200. Если 404 — файла нет на сервере.

## ❌ Заставка не появляется

Проверь имена файлов в `public/assets/intro/`: должны быть `panel1.png`, `panel2.png`, `panel3.png`.

## ❌ Музыка не играет

- Проверь имя: строго `music.mp3` в `public/assets/sounds/`.
- В браузере на компе музыка стартует только после первого клика — это правило безопасности всех браузеров. На заставке тапни, и музыка пойдёт.
- В Telegram должно работать сразу, тоже после первого тапа.

## ❌ Не запускается `npm install`

- Перезагрузи компьютер после установки Node.js.
- Проверь что ты в правильной папке: команда `dir` (Windows) или `ls` (Mac/Linux) должна показать файл `package.json`.

## ❌ На Render деплой не получился

- Зайди в логи (вкладка Logs у твоего сервиса) — там будет ошибка.
- Чаще всего: забыл залить какой-то файл на GitHub. Проверь, что `package.json`, `server.js` и папка `public/` со всеми подпапками — на GitHub.

## ❌ Друг открывает ссылку, но попадает не в твою комнату

Проверь, что в `public/app.js` ты заменил `YourBotName` на свой username бота И обновил файл на GitHub (см. Шаг 4.4).

## ❌ Сервер «засыпает» на Render

Это нормально для бесплатного тарифа. Первая загрузка после простоя — до 30 секунд. Если хочешь чтобы не засыпал — можно настроить бесплатный пинг через сайт **uptimerobot.com** (создай там аккаунт, добавь свою ссылку с Render, ставь пинг каждые 5 минут — и сервер не будет спать).

## 📝 Памятка по правилам твоей колоды

| № | Имя | Сколько штук | Что делает |
|---|---|---|---|
| 1 | Обстоятельства | 5 | Назови карту 2–8. У соперника она? — он выбывает. |
| 2 | Свидетель | 2 | Тайно посмотри карту соперника. |
| 3 | Дуэлянт | 2 | Сравните карты, у кого ниже — выбывает. |
| 4 | Защитник | 2 | До твоего следующего хода ты неуязвим. |
| 5 | Палач | 2 | Любой игрок (можно себя) сбрасывает карту и берёт новую. |
| 6 | Заговорщик | 1 | Обменяйся картами с соперником. |
| 7 | Вдова | 1 | Если на руке также 5 или 6 — обязан сбросить Вдову. Эффекта нет. |
| 8 | Императрица | 1 | Сбросишь по любой причине — мгновенно проигрываешь. |

**Всего 16 карт.** В начале раунда 1 карта откладывается лицом вниз (она не участвует), каждому игроку сдаётся 1 карта. На своём ходу: бери одну, играй одну.
